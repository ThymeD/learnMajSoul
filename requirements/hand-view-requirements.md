## 1. 功能概述

手牌分析页面是雀魂麻将攻略系统的核心功能页面，用于帮助玩家分析当前手牌状态、判断胡牌可能性、计算役种和番数。玩家可以通过拖拽操作管理手牌、副露、摸牌和牌河，系统实时分析并展示分析结果。

---

## 2. 用户场景

| 场景     | 描述                                               |
| -------- | -------------------------------------------------- |
| 手牌编辑 | 玩家通过拖拽或点击的方式从素材区选择牌到各区域     |
| 副露操作 | 玩家在副露区进行吃、碰、杠操作                     |
| 牌局分析 | 玩家点击分析按钮查看当前牌型的胡牌状态、役种和番数 |
| 随机生成 | 玩家需要生成一个符合胡牌条件的牌型进行练习         |
| 牌回收   | 玩家需要将误放的牌退回素材区                       |

### 3.2 数据结构

#### 3.2.1 麻将牌数据结构

```typescript
/** 牌ID格式：
 * - 万：w1~w9 (w=wan)
 * - 筒：b1~b9 (b=bamboo)
 * - 索：s1~s9 (s=string)
 * - 东南西北：d1~d4 (d=direction)
 * - 白中发：z1~z3 (z=zong)
 * - 赤五万/筒/索：rw5/rb5/rs5
 */
type Tile = string

/** 风牌 */
type Wind = 'd1' | 'd2' | 'd3' | 'd4' // 东、南、西、北

/** 三元牌 */
type Dragon = 'z1' | 'z2' | 'z3' // 白、中、发

/** 副露类型 */
type FuluType = 'chi' | 'pon' | 'kan'

/** 副露结构 */
interface Fulu {
  type: FuluType
  tiles: Tile[] // 吃的牌/碰的牌/杠的牌
  from?: number // 来自哪一家（0=上家,1=对家,2=下家）- 吃牌需要
  isOpen?: boolean // 是否明杠（暗杠=false）
}

/** 分析结果 */
interface AnalysisResult {
  isTing: boolean // 是否听牌
  isHu: boolean // 是否胡牌
  tingPai: Tile[] // 听牌列表
  zhenTing: boolean // 是否振听
  han: number // 总番数
  yaku: string[] // 匹配的役种名称列表
  error?: string // 校验错误信息
}
```

#### 3.2.2 素材区数据结构

```typescript
/** 素材区牌数量映射 */
interface SourceTiles {
  [tileId: string]: number // 牌ID -> 剩余数量
}

/** 初始状态示例：
 * {
 *   "w1": 4, "w2": 4, ..., "w9": 4,
 *   "b1": 4, ..., "b9": 4,
 *   "s1": 4, ..., "s9": 4,
 *   "d1": 4, "d2": 4, "d3": 4, "d4": 4,
 *   "z1": 4, "z2": 4, "z3": 4,
 *   "rw5": 4, "rb5": 4, "rs5": 4
 * }
 */
```

### 3.3 交互流程

#### 3.3.1 素材区拖拽流程

```
素材区点击/拖拽
    ↓
检查该牌剩余数量 > 0
    ↓
数量减1，更新UI显示
    ↓
emit select事件，牌添加到目标区域
```

#### 3.3.2 牌移除流程（非素材区）

```
双击非素材区某张牌
    ↓
识别来源区域（手牌/摸牌/副露/牌河）
    ↓
从来源区域移除该牌
    ↓
素材区该牌数量加1（不超过4）
    ↓
emit remove事件
```

#### 3.3.3 副露组合流程

```
拖牌到副露区
    ↓
检查暂存区已有牌
    ↓
判断组合类型：
  - 3张相同牌 → 碰
  - 4张相同牌 → 明杠（初始为明杠，可通过切换按钮变为暗杠）
  - 3张连续牌 → 吃
  - 有碰+1张相同牌 → 明杠
    ↓
形成副露组，移到副露组区域
    ↓
emit fulu事件
```

#### 3.3.4 分析流程

```
点击分析按钮
    ↓
校验牌数量（必须13或14张）
    ↓
校验同种牌不超过4张
    ↓
调用胡牌判定算法
    ↓
调用役种匹配算法
    ↓
展示结果：
  - 错误：显示错误原因
  - 胡牌：显示役种列表、番数
  - 听牌：显示听牌列表、振听状态
  - 未听牌：显示"未听牌"
```

## 4. 功能详细描述

### 4.1 素材选择区（HAND-001）

#### 功能描述

- 展示所有34种麻将牌（不含赤牌共31种，赤牌3种）
- 每种牌初始数量为4张（赤牌与对应普通牌共享4张配额，如：红五万1张 + 普通五万3张 = 五万一共4张）
- 素材区展示顺序：普通5、赤牌、普通6（即赤牌显示在普通5的右侧）
- 支持分类Tab切换（全部、万、筒、索、风、三元、赤）
- 支持搜索牌型

#### 交互方式

- **点击/单击**：素材区点击牌添加到手牌区（如果手牌未满则添加，手牌已满不处理添加）
- **拖拽**：从素材区拖拽牌到其他区域
- **双击**：其他区域的牌移除该牌回到素材区

#### 数量管理

- 拖出或点击后，该牌数量减1
- 数量最小值为0，不能为负数
- 数量为0时，牌显示为禁用状态（灰色）
- 接收其他区域拖回的牌时，数量加1（不超过4）

#### 界面要求

- 每张牌显示剩余数量（右上角数字）
- 数量为0时显示红色背景
- 数量大于0时显示绿色背景

#### 边界条件

- 数量为0时不允许拖出或点击
- 拖回牌时如果已满（4张）则不接受（异常场景，正常不会超过4张牌）

### 4.2 手牌区（HAND-002）

#### 功能描述

- 手牌区：最多容纳13张牌
- 摸牌区：只能容纳1张牌

#### 交互方式

- **拖入**：从素材区、摸牌区、牌河、副露暂存区拖入
- **拖出**：到手牌可拖出到牌河、副露区
- **摸牌替换**：拖入新摸牌时，原摸牌自动移到素材区

#### 数量限制

- 手牌最多13张，超出时提示"手牌已满"
- 摸牌只能1张，超出时原摸牌移到素材区

#### 特殊显示

- 悬停时显示删除按钮，删除图标在牌的右上角

#### 边界条件

- 手牌满13张后不能再拖入
- 摸牌为空时显示"拖入摸牌"占位符

### 4.3 副露区（HAND-003）

#### 功能描述

- 展示已形成的副露组
- 有暂存区用于临时存放牌
- 支持吃、碰、杠操作

#### 副露类型

| 类型              | 组成        | 说明                               |
| ----------------- | ----------- | ---------------------------------- |
| 吃 (chi)          | 3张连续数牌 | 必须包含摸牌或上家打出的牌         |
| 碰 (pon)          | 3张相同牌   | 可直接组成                         |
| 明杠 (kan-open)   | 4张相同牌   | 初始为明杠，可切换为暗杠           |
| 暗杠 (kan-closed) | 4张相同牌   | 初始为明杠，点击切换按钮后变为暗杠 |

#### 暂存区规则

- 拖入牌后先到暂存区
- 暂存区达到3张相同牌自动成碰
- 暂存区达到3张连续数牌自动成吃
- 当暂存区碰、吃冲突时优先碰
- 当暂存区杠、吃冲突时优先杠（标准麻将规则）
- 当暂存区的牌可以和已经碰的牌形成杠时，组成明杠
- 暂存区牌可拖出到其他区域

#### 副露按钮

- 吃按钮：是否禁止吃组合，可点击切换状态
- 碰按钮：是否禁止碰组合，可点击切换状态
- 杠按钮：是否禁止杠组合，可点击切换状态
- 立直后不能进行副露操作，副露区的牌都无法成组
- 副露区未成组剩余牌会在点牌局分析按钮时先回收到素材区再计算牌型

#### 界面显示

- 暗杠：牌背在两边两张，牌背是特殊素材，在素材区不显示，但是实际上代表了该杠牌的牌型，素材区要能正确计算该牌总数，正常素材区该牌可用数量0
- 成组牌：成组的牌右侧有删除按钮，支持删除这组牌，删除的牌回到素材区
- 暗/明杠切换：在杠牌右侧显示删除、切杠按钮，上下布局，支持将杠牌在明杠、暗杠之间切换
- 碰/明杠：正常显示
- 暂存区：黄色背景虚线边框

#### 边界条件

- 立直后不能进行任何副露操作
- 副露后不能立直

### 4.4 牌河（HAND-004）

#### 功能描述

- 展示已打出的牌
- 可拖入任意数量

#### 交互方式

- **拖入**：从素材区、手牌区、摸牌区、副露暂存区拖入
- **拖出**：从牌和拖出到素材区、手牌区、摸牌区、副露暂存区
- **移除**：双击移除或点击牌右上角移除按钮，回到素材区

#### 数量限制

- 无数量限制

#### 界面显示

- 横向排列，自动换行
- 悬停时显示牌右上角的移除符号

#### 边界条件

- 无特殊限制

### 4.5 双击移除（HAND-005）

#### 功能描述

- 非素材区的牌支持双击移除（注意：副露区已成组的牌不支持双击移除，不响应双击）
- 移除的牌返回素材区

#### 适用范围

| 区域       | 是否支持 | 移除行为             |
| ---------- | -------- | -------------------- |
| 素材区     | 否       | -                    |
| 手牌区     | 是       | 整张移除，返回素材区 |
| 摸牌区     | 是       | 整张移除，返回素材区 |
| 副露区     | 否       | -                    |
| 牌河       | 是       | 整张移除，返回素材区 |
| 副露暂存区 | 是       | 整张移除，返回素材区 |

#### 素材区数量更新

- 移除时，素材区对应牌数量加1
- 数量不超过4张

#### 边界条件

- 素材区数量已达4张时，移除的牌不累加（正常不会出现超过4张一样的牌）

### 4.6 清空按钮（HAND-006）

#### 功能描述

- 一键清空所有区域的牌
- 所有牌返回素材区

#### 清空范围

- 手牌区：全部清空
- 摸牌区：清空
- 副露区：全部清空
- 副露暂存区：全部清空
- 牌河：全部清空
- 役种设置：重置为默认值
  - 立直：false
  - 庄家：false
  - 自风：东
  - 场风：东

#### 素材区更新

- 清空后，所有牌数量恢复为4

### 4.7 随机生成（HAND-007）

#### 功能描述

- 生成一个符合胡牌条件的牌型
- 保证生成的牌可以胡牌

#### 生成规则

- 生成13张手牌 + 1张摸牌 = 14张
- 或者生成14张手牌（门清状态）
- 所有牌不超过4张
- 或者生成副露区+手牌+1张摸牌组成胡牌牌型
- 牌河区任意，确保和胡牌规则不冲突，牌总数量符合逻辑

#### 役种符合性

- **必须满足**：生成的牌型必须是胡牌牌型
- **应尽量多样**：随机选择不同的胡牌牌型

#### 边界条件

- 确保牌池充足
- 确保不超过4张相同牌

### 4.8 分析功能（HAND-008）

#### 功能描述

- 分析当前牌局的胡牌可能性
- 计算役种和番数

#### 分析条件

- 必须有13或14张牌（13手牌+1摸牌 或 14手牌 或 副露区+手牌满足胡牌要求）
- 同种牌不能超过4张
- 注意副露区的杠牌，会增多1张牌，每个杠牌会将胡牌牌型增加一张
- 有副露（含暗杠）时不能立直，副露区有牌则不能立直胡牌

#### 结果展示

| 状态   | 显示内容                             |
| ------ | ------------------------------------ |
| 错误   | 错误原因（用红色alert显示）          |
| 胡牌   | 状态标签（绿色）、役种列表、番数     |
| 听牌   | 状态标签（黄色）、听牌列表、振听状态 |
| 未听牌 | 状态标签（灰色）、"未听牌"           |

#### 役种展示要求

- 显示所有满足条件的役种名称
- 显示总番数
- 役种名称之间上下布局，每个名称一行显示

#### 错误提示（用户能理解的语言）

| 错误情况                                                | 提示信息                                                  |
| ------------------------------------------------------- | --------------------------------------------------------- |
| 同种牌超过4张                                           | "牌型不合法：{牌名}已有{数量}张，同种牌不能超过4张"       |
| 牌数不足14张                                            | "牌数不足，手牌+摸牌+副露 需要14张手牌，当前只有{数量}张" |
| 牌数超过14张且不符合逻辑（杠牌会加1张，其他不符合逻辑） | "牌数过多，最多{数量}张，当前有{数量}张"                  |

#### 役种计算规则

- 结合役种设置（立直、庄家、自风、场风）综合判断
- 门前清役种需要无副露
- 副露后役种需要有过副露

### 4.9 役种设置（HAND-009）

#### 功能描述

- 设置立直状态
- 设置庄家/闲家
- 设置自风
- 设置场风

#### 设置项

| 设置项 | 类型     | 默认值 | 说明             |
| ------ | -------- | ------ | ---------------- |
| 立直   | checkbox | false  | 门清状态下可勾选 |
| 庄家   | checkbox | false  | 是否为庄家       |
| 自风   | select   | 东     | 东南西北选择     |
| 场风   | select   | 东     | 东南西北选择     |

#### 联动规则

- 有副露（含暗杠）时，立直checkbox禁用
- 勾选立直后，不能进行副露操作

## 6. 验收标准

### 6.1 素材选择区（HAND-001）

- [ ] 每种牌初始显示数量为4
- [ ] 点击牌后，数量减1
- [ ] 拖出牌后，数量减1
- [ ] 数量为0时，牌显示为禁用状态
- [ ] 拖回牌后，数量加1
- [ ] 数量最大为4，不会超过

### 6.2 手牌区（HAND-002）

- [ ] 手牌最多13张，超出提示"手牌已满"
- [ ] 摸牌只能1张
- [ ] 拖入新摸牌时，原摸牌移到素材区
- [ ] 手牌满13张后不能再拖入

### 6.3 副露区（HAND-003）

- [ ] 3张相同牌自动成碰
- [ ] 4张相同牌自动组成明杠
- [ ] 碰+第4张自动发明杠
