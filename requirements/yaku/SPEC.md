# 役种一览功能需求规格

## 1. 功能概述

役种一览是雀魂麻将攻略系统的核心功能页面，用于展示和学习麻将游戏中的全部役种（35+种）。用户可以通过筛选、搜索、熟练度管理等功能，高效记忆役种名称、番数和牌型。

## 2. 用户场景

| 场景 | 描述 |
|------|------|
| 役种查询 | 玩家想不起来某个役种的番数或牌型，通过筛选或搜索快速定位 |
| 役种记忆 | 玩家通过"已胡次数"功能记录自己已经会胡的役种，帮助查漏补缺 |
| 番数筛选 | 玩家想复习"满贯"以上的役种，按番数筛选查看 |
| 分类学习 | 玩家想专注学习"门前清"限定的役种，按分类筛选 |

## 3. 功能清单

### 3.3 数据结构

```typescript
interface Yaku {
  id: string              // 唯一标识，如 "reach", "tanyao"
  name: string            // 役种名称，如 "立直"
  han: number             // 番数：1-6, 5(满贯), 8(役满), -2(双倍役满), -3(流局)
  category: '无限制' | '门前清' | '副露后'  // 分类
  desc: string            // 役种描述
  tiles: string[]         // 牌型数组，如 ["w1", "w2", "w3", ...]
  splitAt?: number[]      // 分割位置数组，用于显示听牌间隔
  mastery?: number        // 已胡次数（运行时）
  isHu?: boolean          // 是否为胡牌役
  isEffectOnly?: boolean  // 是否仅为效果类（如宝牌）
  isDealerOnly?: boolean  // 是否庄家限定
  isNonDealerOnly?: boolean // 是否子家限定
  note?: string           // 备注说明
}
```

## 4. 验收标准

### 4.1 番数筛选

- [ ] 点击"一番"tab，列表只显示 han=1 的役种
- [ ] 点击"役满"tab，列表只显示 han=8 的役种
- [ ] 点击"双倍役满"tab，列表只显示 han=-2 的役种
- [ ] 点击"流局"tab，列表只显示 han=-3 的役种
- [ ] 点击"全部牌型"，显示全部役种（不受番数限制）
- [ ] 切换番数后，分类筛选自动重置

### 4.2 分类筛选

- [ ] 分类选项根据当前番数动态生成（如的一番只有"无限制"和"门前清"）
- [ ] 选择"门前清"，列表只显示 category="门前清" 的役种
- [ ] 选择"无限制"，列表同时显示"无限制"和"副露后"（副露后番数减1但不是门前清）
- [ ] 选择"不是役"，列表只显示 isEffectOnly=true 的役种（如宝牌）

### 4.3 搜索功能

- [ ] 输入"立直"，列表只显示名称包含"立直"的役种
- [ ] 输入"幺九"，列表显示描述中包含"幺九"的役种
- [ ] 点击"清除"按钮，清空搜索词并恢复筛选结果
- [ ] 搜索框空时，显示当前筛选条件的结果

### 4.4 熟练度管理

- [ ] 点击"已胡"数字，数字+1，有绿色+1动画
- [ ] 长按"已胡"数字区域800ms后，开始连续减少
- [ ] 长按卡片区域（不是数字区域）800ms后，开始连续增加
- [ ] 数字减少到0后不再减少
- [ ] 长按时显示进度条动画
- [ ] 页面刷新后，已胡次数保持不变

### 4.5 持久化存储

- [ ] 第一次访问，"已胡"显示0
- [ ] 点击+1后刷新页面，数字保持+1
- [ ] 手动清除 localStorage，数据重置为0
- [ ] 筛选条件（番数、分类）刷新后保持

### 4.6 牌型展示

- [ ] 每张役种卡片底部显示对应牌型
- [ ] 牌型使用 MahjongTile 组件渲染
- [ ] 听牌位置（splitAt）有分割间隔显示
- [ ] 役种无牌型时（如四家立直）不显示牌型区域

### 4.7 导航侧边栏

- [ ] 右侧显示当前筛选结果的役种名称列表
- [ ] 列表项数量与主列表一致
- [ ] 点击列表项，主列表滚动到对应位置
- [ ] 当前选中的役种在导航中高亮

## 5. 非功能需求

### 5.1 性能要求

- 页面加载时间 < 1秒
- 筛选/搜索响应时间 < 100ms
- 长按连续操作间隔 200ms

### 5.2 兼容性要求

- 支持 Chrome、Edge、Firefox 最新版本
- 支持 1920x1080 及以上分辨率
- 移动端可正常浏览（响应式布局）

### 5.3 数据完整性

- 役种数据共 35+ 种（含役满、Double役满、流局）
- 所有役种必须有 name、han、category、desc、tiles 字段
- 牌型数据与实际麻将牌一致

### 5.4 可维护性

- 役种数据独立存储在 yaku-config.json，便于扩展
- 组件样式使用 Scoped CSS，隔离性好
- TypeScript 严格类型检查
