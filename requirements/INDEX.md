# 项目需求统一索引

## 快速使用（30秒）
1. 先看本页“需求清单”，确认目标模块与当前状态。
2. 进入对应需求文档查看验收标准与边界条件。
3. 开发前后同步更新：本索引状态 + 对应需求文档状态。
4. 若与现有需求冲突，先登记“变更中”并二次确认，再实施。

## 使用规则
1. 所有功能需求文档必须在本索引登记后才视为“项目需求”。
2. 任何新需求实现后，必须同步刷新对应文档状态（已实现/已验证/待验证）。
3. 当用户提出需求调整，与既有需求冲突时，先在本索引登记“变更中”，完成二次确认后再落地实现。
