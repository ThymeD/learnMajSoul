# 业务项目发布 / 部署确认清单

适用于：**本仓库作为「雀魂攻略」等业务应用上线**（静态托管、内网镜像、应用商店内嵌 WebView 等），需要明确 **交付/数据管理** 与 **生产环境** 的差异。

> 与「管理系统拆出去」无关的日常发版，以本清单为主；PM 套件单独迁移见 [管理系统移植确认清单](./checklist-pm-suite-migration.md)。

---

## 1. 功能开关与暴露面

- [ ] **`src/config/features.ts`** 中 `deliveryManagement` 是否为生产预期（`false` 则菜单隐藏交付/数据管理，减少攻击面与无效请求）。
- [ ] 若关闭交付功能：确认 **无** 其它入口链到 `/delivery`、`/data-management`（深链、书签、文档链接）。
- [ ] 产品/运营是否需在对外文档中 **不写** 内部管理路由。

---

## 2. 构建产物与本地 API

- [ ] **生产构建**（`npm run build`）**不包含** Vite 中间件；`/__pm_api/*` **仅在 `npm run dev` 时可用**。
- [ ] 若用户误用 **纯静态部署** 打开站点：交付/数据管理页的读写会失败；是否在帮助文档中说明「管理功能需本地开发环境」或后续独立后端。
- [ ] `npm run build` **不会**执行 `predev`（仅 `npm run dev` 前执行）；CI 里若误加 `predev` 需确认是否接受对 `.pm-center` 的写入。

---

## 3. 密钥、路径与隐私

- [ ] 前端 bundle **不包含** GitHub Token、SSH 私钥；PM 相关操作依赖本机 Git 凭据与环境（与当前设计一致）。
- [ ] 发布物中若有 **Source Map** 外发：注意是否暴露 `projectKey`、仓库路径等（通常风险低，按公司策略决定）。

---

## 4. 性能与首屏

- [ ] 交付/数据管理为 **懒加载路由**；关闭 `deliveryManagement` 时相关 chunk 不应被主路径请求（除非被预取；可按需检查）。

---

## 5. 监控与回滚

- [ ] 生产若 **不** 启用交付功能：无需为 `__pm_api` 配置代理。
- [ ] 若曾误开管理页给公网用户：回滚版本或关 feature flag 即可。

---

## 6. 与「管理系统移植」的边界

- 业务发版 **不要求** 迁移 `src/pm`、脚本或 `.pm-center`；仅当要把 **PM 能力带到别的宿主仓库** 时，再使用 [管理系统移植确认清单](./checklist-pm-suite-migration.md)。
