# 项目交付与数据管理 — 使用说明

面向：**用本应用管理「AI 辅助开发」交付事项与进度数据的用户**（个人项目、多设备均可）。

> **双仓概念**见 [管理数据仓库概览](./pm-data-repo-overview.md)。开发与命令行细节见 [项目管理关联与备份](./pm-linking.md)；分支策略见 [PM 数据仓分支约定](./pm-data-repo-branches.md)。

---

## 1. 两个页面分别做什么

| 页面 | 路由 | 用途 |
|------|------|------|
| **项目交付管理** | `/delivery` | 日常主战场：交付项、看板、状态流转、模板导入、与「用户确认 / AI 提交」等流程相关操作。 |
| **数据管理** | `/data-management` | 运维与排障：业务仓库 ↔ 管理数据仓库是否关联、工作模式、环境探测、**多设备下的**同步/备份/分支、日志裁剪等。 |

原则：**平时在交付页工作**；换机、重装、同步异常、或首次配置双仓时，再打开数据管理。

---

## 2. 使用前需要满足什么

- 使用 **开发模式**（`npm run dev`）时，本地项目文件 API（`/__pm_api/*`）由 Vite 提供；交付数据写入「管理数据仓」目录下的 JSON。
- 若启用交付功能，需在菜单中能看到「项目交付管理」「数据管理」（由项目 `featureFlags` 控制）。
- **便携中心**与 **双仓映射**未就绪时，数据管理页会提示先完成关联；可按 [pm-linking.md](./pm-linking.md) 用 `npm run pm:init` 等命令初始化。

---

## 3. 工作模式：本地单机 vs 多设备同步

在 **数据管理 → ① 模式与路径** 中切换：

- **本地单机**：降低打扰；总览会**折叠**「分支 / 同步 / 备份」等强运维区块，编号自动紧凑排列。远端同步与备份可按阶段再处理。
- **多设备同步**：适合多台电脑、或与远端 GitHub 对齐；总览展示完整卡片，顶栏也可能出现「落后远端 / 领先远端」等提醒。

切换成功后会 **自动刷新页面**，使总览与顶栏与最新模式一致。

---

## 4. 顶栏全局提醒

在非数据管理页时，若需要处理，顶部可能出现：

- **必须处理**：工作模式未确认（例如检测到远程仓库变化）。
- **多设备模式下**：数据仓落后远端、或本地有提交未推送等（可稍后提醒）。

进入 **数据管理** 页时，顶栏不重复展示同类提醒，避免重复。

---

## 5. 页面与命令行对照

| 目的 | 应用内 | 命令行（可选） |
|------|--------|----------------|
| 看关联与同步状态 | 数据管理总览、状态看板 | `npm run pm:link-status` |
| 备份管理数据到远端 | 数据管理「备份」相关（多设备模式） | `npm run pm:backup` |
| 初始化便携中心与映射 | 按界面提示 | `npm run pm:init` |

详细行为以 [pm-linking.md](./pm-linking.md) 为准。

---

## 6. 常见问题（简）

- **交付页数据存在哪？**  
  由 `projectKey` 关联到管理数据仓目录下的 `*-items.json` 等文件（见数据管理中的路径与上下文说明）。
- **为什么单机模式看不到同步/备份卡片？**  
  为减少干扰，已折叠；切换到「多设备同步」即可展开（配置不丢）。
- **构建后的静态站点** 若未配置等价后端，项目文件 API 不可用；交付/数据管理依赖本地 dev 或后续独立部署的后端。
