# 手牌分析 - 功能使用文档

## 1. 功能介绍

手牌分析是雀魂麻将攻略系统的核心模块之一，用于分析和学习麻将的手牌状态。该功能提供手牌输入、胡牌判定、听牌分析、役种匹配等功能，帮助玩家理解牌型结构和提升打牌策略。

### 主要功能特性

- **手牌输入**：通过点击或拖拽方式输入手牌
- **胡牌判定**：判断当前牌型是否满足胡牌条件
- **听牌分析**：分析听牌范围（能胡哪些牌）
- **振听判定**：检测是否处于振听状态
- **副露管理**：支持吃、碰、杠操作
- **牌河记录**：记录打出的牌
- **役种匹配**：匹配可成的役种
- **番数计算**：计算总番数

## 2. 使用说明

### 2.1 访问入口

在应用导航栏中找到「手牌分析」入口，点击即可进入功能页面。

### 2.2 手牌输入

#### 方式一：点击选择

1. 在左侧素材选择区点击想要的牌
2. 牌会自动添加到右侧手牌区
3. 每种牌最多添加4张

#### 方式二：拖拽

1. 从左侧素材区拖拽牌到右侧手牌区
2. 松开鼠标即可添加

#### 方式三：随机生成

点击「随机生成」按钮，系统会自动生成一手随机牌型。

### 2.3 摸牌设置

拖拽一张牌到摸牌区域，可以设置摸牌。

### 2.4 副露操作

#### 吃牌

1. 点击「吃」按钮
2. 选择可以组成顺子的3张牌
3. 确认后添加到副露区

#### 碰牌

1. 点击「碰」按钮
2. 选择3张相同的牌
3. 确认后添加到副露区

#### 杠牌

1. 点击「杠」按钮
2. 选择4张相同的牌
3. 确认后添加到副露区

### 2.5 牌河操作

#### 打牌

从手牌区拖拽一张牌到牌河区域，即可打出这张牌。

#### 悔牌

从牌河区域拖拽一张牌回手牌区，即可回收这张牌。

### 2.6 役种设置

#### 庄家

勾选「庄家」设置当前为庄家。

#### 自风/场风

选择自风和场风，用于计算役牌番数。

#### 立直

勾选「立直」设置立直状态。

> 注意：副露后无法立直，立直后无法副露。

### 2.7 分析结果

点击「分析」按钮，系统会分析当前牌型并显示：

| 项目 | 说明              |
| ---- | ----------------- |
| 状态 | 胡牌/听牌/未听牌  |
| 听牌 | 能胡的牌列表      |
| 振听 | 是否振听（是/否） |
| 番数 | 总番数            |
| 役种 | 匹配的役种列表    |

## 3. 核心概念

### 3.1 胡牌条件

日本麻将胡牌需要满足：

- 4组面子（顺子、刻子或杠子）+ 1组雀头
- 正常手牌14张（13张手牌 + 1张摸牌）

### 3.2 面子类型

| 类型 | 说明              | 示例         |
| ---- | ----------------- | ------------ |
| 顺子 | 连续的3张同花色牌 | 1万2万3万    |
| 刻子 | 3张相同的牌       | 4筒4筒4筒    |
| 杠子 | 4张相同的牌       | 5索5索5索5索 |

### 3.3 雀头

2张相同的牌，作为胡牌的雀头。

### 3.4 听牌

差1张牌即可胡牌的状态。系统会分析并显示所有能胡的牌。

### 3.5 振听

听牌后，自己打过的牌（含摸牌）不能再荣和。系统会根据牌河自动判定是否振听。

### 3.6 立直

门前清状态下听牌时，可以宣言立直。立直后：

- 无法副露
- 无法换听牌
- 牌河公开

### 3.7 特殊役满

| 役种           | 说明                             |
| -------------- | -------------------------------- |
| 七对子         | 7组不同的对子                    |
| 国士无双       | 13种幺九牌各1张 + 任意1张幺九    |
| 国士无双十三面 | 国士无双的十三面听牌状态         |
| 九莲宝灯       | 同种数牌1112345678999 + 任意同种 |
| 纯正九莲宝灯   | 九莲宝灯的九面听牌状态           |

## 4. 调试方法

### 4.1 浏览器控制台 API

项目在全局暴露了 `handUtils` 对象，可在浏览器控制台中直接调用：

```javascript
// 获取手牌 store
const store = handUtils.getStore()

// 随机生成一手牌
store.randomHand()

// 添加一张牌
store.addTile('w1')

// 设置摸牌
store.setDrawTile('w2')

// 执行分析
store.analyze()

// 查看分析结果
console.log(store.analysis)
// { isTing: true, isHu: false, tingPai: ['w3'], ... }
```

## 5. 常见问题

### Q1: 为什么无法添加某张牌？

检查该种牌是否已经达到4张上限。每种牌最多只能有4张。

### Q2: 为什么无法立直？

立直需要满足以下条件：

- 门清状态（无副露）
- 听牌状态

如果已经有副露，或者还未听牌，则无法立直。

### Q3: 为什么显示"未听牌"？

手牌需要满足以下条件才能听牌：

- 手牌13张 + 摸牌1张 = 14张
- 牌型可以分割为4组面子 + 1组雀头

### Q4: 什么是振听？

振听是指：听牌后，自己打过的牌（含摸牌）不能再荣和。如果听牌列表中有牌在牌河里，系统会显示"振听：是"。

### Q5: 番数如何计算？

番数计算规则：

1. 累加所有匹配的役种番数
2. 副露后1-2番役种减1番（门前清役种除外）
3. 役满（8番以上）单独处理
