# 手牌分析页面需求描述

> ⚠️ **重要**：以下需求在复用时需要让用户主动确认！

## 页面概述

手牌分析页面用于分析麻将手牌是否听牌/胡牌，支持役种计算和番数统计。

## 核心功能

### 1. 素材选择区

- 显示所有麻将牌（万/筒/索/字/赤）
- 每种牌默认4张
- 右上角显示剩余数量
- 支持搜索过滤
- **拖拽到其他区域时减少数量**

### 2. 手牌区

- 最多13张牌
- 显示听牌标记（听牌时显示听的牌和剩余张数）
- 支持点击/拖拽添加牌到其他区域
- 支持从其他区域接收牌

### 3. 摸牌区

- 最多1张牌
- 支持点击选手牌
- 支持拖拽到其他区域

### 4. 副露区

- 支持吃（chi）、碰（pon）、杠（kan）
- 暂存区逻辑：
  - 拖入3张相同牌 → 成碰
  - 拖入4张相同牌 → 成暗杠
  - 已有碰 + 拖入1张 → 成明杠
  - 暂存区的牌可与新拖入的牌形成吃（优先于杠）
- 删除副露时牌回到素材区

### 5. 牌河区

- 展示打出的牌
- 支持拖拽到其他区域

### 6. 分析功能

- 检查同种牌不超过4张
- 检查手牌+摸牌=13或14张
- 计算役种和番数

## 拖拽互通需求（核心难点）

### 拖拽源和目标

| 源 \ 目标  | 素材区 | 手牌区 | 摸牌区 | 副露暂存区 | 牌河区 |
| ---------- | ------ | ------ | ------ | ---------- | ------ |
| 素材区     | -      | ✅     | ✅     | ✅         | ✅     |
| 手牌区     | ✅     | ✅      | ✅     | ✅         | ✅     |
| 摸牌区     | ✅     | ✅     | -      | ✅         | ✅     |
| 副露暂存区 | ✅     | ✅     | ✅     | -          | ✅     |
| 牌河区     | ✅     | ✅     | ✅     | ✅          | -      |

### 关键实现要点

1. **dropEffect 效果**
   - 素材区 → 其他：copy（复制）
   - 其他区 → 其他：move（移动）

2. **事件处理**
   - 各区域使用 `stopPropagation()` 防止事件冒泡
   - 但素材区不能使用 `stopPropagation()`，否则会阻止拖到其他区域

3. **数量同步**
   - 素材区：内部维护 sourceTiles 状态
   - 拖出时立即减少 sourceTiles
   - 拖回时增加 sourceTiles

4. **DOM 结构**
   - 拖拽区域需要能接收 drop 事件
   - 需要设置 dragover 事件处理 dropEffect

## 技术实现参考

### 拖拽数据结构

```typescript
// 拖拽时传递的数据
dataTransfer.setData('text/plain', tileId) // 牌ID
dataTransfer.setData('source', 'hand') // 来源区域
dataTransfer.effectAllowed = 'copy' | 'move' // 效果
```

### 来源区域标识

- `source` - 素材区
- `hand` - 手牌区
- `draw` - 摸牌区
- `fulu-temp` - 副露暂存区
- `river` - 牌河区

### 拖拽效果设置

```javascript
// dragover 事件中
if (e.dataTransfer) {
  const source = e.dataTransfer.getData('source')
  e.dataTransfer.dropEffect = source === 'source' ? 'copy' : 'move'
}
```
