# 上下文效率与文件规模规则

## 目标
在不影响功能交付质量的前提下，降低上下文冗余与 token 消耗，让 Cursor 在长周期迭代中保持稳定效率。

## 读取策略（强制）
1. 先读索引和协作文档，再读代码：
   - `collaboration.md`
   - `requirements/INDEX.md`
   - 对应模块需求文档
2. 对大文件优先“按范围读取”，避免整文件读取：
   - 先用搜索定位符号/区域
   - 再读取目标区段
3. 单次任务最多深入 2-3 个核心文件；若超出则先写入任务拆分并分步执行。

## 文件规模建议（软上限，新增代码遵循）
- 页面入口文件（`views/**/index.vue`）：
  - `<script setup>` 建议 <= 250 行
  - 文件总行数建议 <= 700 行
- store 文件：建议 <= 500 行
- 工具算法文件：建议 <= 600 行
- 需求文档：建议拆成“总览 + 模块文档 + 验收清单”

## 重构触发条件（不改业务语义）
满足任一条件，进入“重构任务池”：
1. 文件总行数 > 900
2. 同文件承担超过 3 个独立职责（状态、交互、算法、渲染）
3. 同类逻辑出现 2 次以上复制粘贴

## 非功能重构边界
- 允许：拆分 composable、抽公共函数、文档索引化、测试补充
- 禁止：改变现有交互语义、变更业务规则、修改 API 行为
- 要求：重构前后必须通过既有测试（至少 `npm run test:run`）
