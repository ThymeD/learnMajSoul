# 问题7验收测试记录：暗杠牌背素材间距

## 测试日期

2026-03-19

## 分支

fix/batch-3-problems-7-9

## 验收标准

- 暗杠的4张牌之间有间距
- 间距与碰的牌（或吃牌）的间距一致

## 测试步骤

1. 启动开发服务器（npm run dev）
2. 从素材区拖4张1万到副露暂存区
3. 点击"暗杠"按钮形成暗杠
4. 观察暗杠4张牌的间距

## 测试结果

### 暗杠牌位置数据

| 牌序号 | 素材类型      | left | right | width |
| ------ | ------------- | ---- | ----- | ----- |
| 1      | bg.jpg (牌背) | 731  | 771   | 40    |
| 2      | w1.jpg (正面) | 771  | 811   | 40    |
| 3      | w1.jpg (正面) | 811  | 851   | 40    |
| 4      | bg.jpg (牌背) | 851  | 891   | 40    |

### 间距计算

- 牌1到牌2：771 - 771 = **0px**
- 牌2到牌3：811 - 811 = **0px**
- 牌3到牌4：851 - 851 = **0px**

### 结论

暗杠4张牌之间**没有间距**，是紧密排列的。每张牌的右边界与下一张牌的左边界完全重合。

## 附加发现

1. 按钮显示"明杠"而非"暗杠"，但这可能是从素材区拖入4张牌的正常行为
2. 暗杠显示为2张牌背+2张正面，而不是全部牌背
